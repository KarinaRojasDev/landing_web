NeoFichaje – Aplicación Android

Aplicación móvil desarrollada para optimizar el control horario y la gestión de empleados en pequeñas y medianas empresas.

Descripción del Proyecto

NeoFichaje es una aplicación Android diseñada para digitalizar el registro de jornada laboral, la gestión de vacaciones y la administración documental de empleados. Permite a los trabajadores registrar entradas y salidas, solicitar permisos y consultar documentación personal, mientras que los administradores pueden supervisar y gestionar su equipo de forma remota.

Tecnologías Utilizadas

  • Android Studio: Entorno de desarrollo
  • Kotlin: Lenguaje principal
  • XML: Diseño de interfaces
  • Firebase Authentication: Registro e inicio de sesión seguro
  • Firestore: Base de datos NoSQL en la nube
  • Firebase Cloud Storage: Almacenamiento de documentos
  • Google Sign-In: Autenticación con cuenta de Google
  • MaterialCalendarView: Selección avanzada de fechas

Roles de Usuario

Empleado / Técnico

  • Registro de entradas y salidas
  • Solicitudes de vacaciones y permisos
  • Visualización de documentos personales

Administrador / Empresario

  • Alta y gestión de empleados
  • Aprobación de solicitudes
  • Subida de contratos y nóminas
  • Supervisión de asistencia

Funcionalidades Destacadas

  • Fichaje con geolocalización
  • Registro y login con correo electrónico y Google
  • Consulta y descarga de contratos y nóminas
  • Gestión de vacaciones y permisos
  • Sistema de notificaciones internas
  • Historial de actividad por usuario

Flujo de Uso

  1. El usuario se registra o inicia sesión.
  2. Accede al panel correspondiente según su rol.
  3. Los empleados registran su jornada o gestionan solicitudes.
  4. Los administradores supervisan, aprueban y gestionan documentos.

LiterAlura – Aplicación Backend en Java (Spring Boot)

Descripción del Proyecto

LiterAlura es una aplicación en Java con Spring Boot que permite buscar libros usando la API de Gutendex, registrar información de libros y autores en PostgreSQL, y consultar los datos por distintos criterios como idioma, año de nacimiento de los autores y descargas. El proyecto integra API REST, JPA/Hibernate, y un menú interactivo por consola, combinando gestión de datos locales con información de una API externa.

Funcionalidades

  • Buscar libros en la API de Gutendex y registrarlos en la base de datos.
  • Listar libros registrados.
  • Listar autores registrados.
  • Filtrar autores vivos en un año específico.
  • Filtrar libros por idioma.
  • Mostrar información de descargas y detalles de cada libro.

Tecnologías utilizadas

  • Java 17+
  • Spring Boot 3
  • Spring Data JPA / Hibernate
  • PostgreSQL
  • Gutendex API
  • Maven

Instalación y configuración

  1. Clonar el repositorio: Ver repositorio en GitHub
  2. Acceder a la carpeta: cd literalura
  3. Configurar las variables de entorno para la base de datos (ejemplo en PowerShell):
    • DB_HOST="localhost"
    • DB1_NAME="LiterAlura"
    • DB_USER="postgres"
    • DB_PASSWORD="tu_contraseña"
  4. Ejecutar la aplicación: mvn spring-boot:run
  5. Interactuar con la aplicación mediante el menú de consola.

Estructura del proyecto

  • model/ → Clases de entidad Libro y Autor.
  • repository/ → Repositorios JPA para acceder a la base de datos.
  • service/ → Servicios que consumen la API y manejan la lógica de negocio.
  • record/ → Records para mapear JSON de la API.
  • main/Main.java → Menú de consola interactivo.
  • LiterAluraApplication.java → Clase principal que arranca la aplicación.

API utilizada

Gutendex API: para obtener información de libros y autores.
